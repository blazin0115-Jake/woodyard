**场景：** Aromix 下载了 Call of Duty 盗版游戏，我帮忙安装。

## 2026-06-25: PowerShell Substring 重复炸 + 无脑重复扫

### 问题 1: Substring 炸 3 次

**场景：** 写 `$_.Exception.Message.Substring(0,80)` 截断错误消息，结果消息 < 80 字时炸了。

**犯的错误：**
- 炸了 3 次没修代码，只是换工具绕路

**正确的做法：**
- 第一次炸 → 检查代码，改安全写法
- 安全写法：
  ```powershell
  if ($_.Exception.Message.Length -gt 80) {
      $_.Exception.Message.Substring(0,80)
  } else {
      $_.Exception.Message
  }
  ```
- 或直接输出全文不截断
- **原则：** 同一个 bug 不炸第二次

### 问题 2: 无脑重复扫 | 绕弯

**场景：** 同样的 endpoint 在不同工具间反复试：
```
curl → 被挡  | Python → 被挡  | PowerShell → 炸
→ 换个写法重跑 | → 又炸 | → 换 Invoke-WebRequest | → 还是炸
```
每次只换工具不分析失败原因。Cloudflare 挡 curl 也会挡 Python。

**正确的做法：**
1. 第一次失败 → 看错误原因
   - Cloudflare (HTML 响应) → 需要浏览器，不需要换 curl 变体
   - app 403 (JSON 响应) → 需要 auth，换什么工具结果一样
   - 语法/工具 bug → 修代码，不是换工具
2. **分析 > 重试。** 失败一次就停下来想为什么，而不是无脑重试。
3. 同样的 endpoint 用不同工具测最多 2 次。第 2 次还不行 → 停下来分析。

**原则：** 失败一次想原因，不想原因不重试。
